# Documentation

## Standards

- Use sentence case for headings (not title case)
- Include table of contents for documents longer than 5 sections
- Link to related documents at the end of each file

## Organization

- Feature-specific documentation goes in subdirectories
- Progress tracking documents stay in `/docs/progress/`
- Architecture and technical patterns in `/docs/project/`

## Update requirements

- Update `context_recovery.md` at the start and end of each session
- Keep `todo_list.md` current with active tasks and progress
- Document version changes and migrations in relevant files
